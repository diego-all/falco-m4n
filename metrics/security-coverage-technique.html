Security Coverage

It refers to how well Falco's rules address potential attack vectors and cover different attack surfaces.

Coverage can be measured in terms of:

Security coverage based on MITRE tactics

Falco provides rules specifically designed to detect suspicious security behaviors in Kubernetes, Linux, and the cloud. This indicator is based on the total number of MITRE tactics covered by Falco rules. It is important to note that it is based on the MITRE Enterprise framework, with 12 relevant tactical categories selected.

91.66%
Target 2025:

The main objective is to address all the tactical categories defined by MITRE to achieve 100% tactical coverage.

Address the tactical categories with the fewest rules or those that are most critical in terms of cloud cybersecurity risks.

Security coverage based on detection surface

Allie Mellen describes the "detection surface," which, according to Forrester, is "the type of IT asset where attacker activity is detected."

The selected surfaces include: filesystem, database, shell, container, users, network, and processes.

Target 2025:

Achieve incorporating at least 2 rules per detection surface.

Try to ensure that no detection surface is below 10%.
