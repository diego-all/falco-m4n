ThirdParty Rules

Below is the list of ThirdParty rules contributed by the Falco runtime security community and other sources. While these rules may not be the most precise, they help enhance detection coverage in certain critical scenarios where official rules from the vendor are not yet available.

MELI ThirdParty Rules Baseline

  • Backdoored library loaded into SSHD
  • PTRACE anti-debug attempt
  • PTRACE attached to process
  • Find AWS Credentials
  • Linux CGroup Container Escape Vulnerability (CVE-2022-4092)
  • Execution from /dev/shm
  • Suspicious Java Child Processes
