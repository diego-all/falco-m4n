Custom Rules

Custom Falco rules are designed to enhance detection coverage for MELI's clusters and workloads, addressing critical security needs. MELI's vast and complex infrastructure demands broader detection across surfaces like filesystem, database, shell, container, users, network, and processes. Default rules alone are not enough. Below, you will find a list of custom rules written to mitigate cybersecurity risks that could impact the organization and are not covered by the vendor's default rules.

Rule 1: Launch memory dump process in Container [custom] ℹ Haz clic para desplegar más información
Description ℹ Haz clic para desplegar más información
Detects the execution of processes related to memory dumping. Memory dumping can be used to analyze a process's state, including the extraction of sensitive information. The tools typically used for this purpose include dd, coredumpctl, gcore, memdump, dump2fs, and dotnet-dump.
Relevance ℹ Haz clic para desplegar más información
Memory dumps can contain sensitive information, such as credentials, API keys, or user data, that an attacker could extract to compromise the system's security. Detecting their execution allows for the identification of possible data extraction attempts or malicious activities within the cluster.
Environment: Production

95%

Rule 2: Connect to NGROK Service [Custom] ℹ Haz clic para desplegar más información
Description ℹ Haz clic para desplegar más información
Detects connections to the NGROK service.
Relevance ℹ Haz clic para desplegar más información
Connections to NGROK can create hidden tunnels that expose internal services to the internet, potentially allowing attackers to establish command and control (C2) channels. Detecting these connections helps identify possible malicious activities or data exfiltration attempts, protecting the security of the cluster.

90%
